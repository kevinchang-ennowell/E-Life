# 系統分析書 - PIC合作活動分析書

# 模組目的與活動樣態
## 目的

- 讓樺康與統一合作規劃活動，並在住戶 App E-Life 露出活動入口（Banner/活動卡）。
- 住戶完成「E-Life 與統一集團好住家 App 的帳號綁定」後，住戶即可獲得 Open Point 點數作為綁定獎勵。
- 在活動期間內，只要住戶「已綁定」且在 好住家 App 有消費行為，就能獲得分潤。
- 後台可管理活動與導入組織、案場，並追蹤綁定成果、消費成果、分潤報表。
- 導入與分潤歸屬以「案場」為單位（哪個社區導入，成效與分潤就歸哪個社區）。

## 活動成立條件
- 住戶在 E-Life 看到活動 → 點「綁定」→ 綁定成功領 Open Point
- 住戶在 好住家 消費 → 樺康取得分潤（活動期間且已綁定才算）

# 使用者與角色
## 樺康
- 合作營運：建立統一集團合作活動、設定入口素材、指定哪些案場導入、查看成效
- 行銷：曝光率、綁定轉換率、素材/文案版本、活動成效敘事
- 財務：分潤報表（按案場/期間）、必要時匯出彙整
- 客服/營運支援：協助住戶綁定問題、查綁定狀態

## 社區/組織端

- 物管公司/社區管理者：通常只需知道「本站是否導入」與「成效摘要」，不需要能改活動規則

## 住戶端

- 住戶：在 E-Life 綁定好住家、領 Open Point、並在好住家消費

# 功能說明
## 合作廠商管理
- 建立合作廠商資料（窗口、合作狀態）
- 紀錄合作模式(合作活動時間段、分潤規則紀錄)
- 紀錄合作條款摘要（方便內部查詢）

3.2 活動管理（E-Life × 好住家 活動）

建立活動：活動名稱、活動期間、活動說明、注意事項

設定活動入口素材：E-Life 的 Banner/活動卡圖片、標題、引導文案

活動狀態：草稿 / 進行中 / 暫停 / 已結束
-（可選）素材版本管理：不同版本素材便於行銷討論

3.3 案場導入管理（以 Site 為單位）

選擇哪些 **案場 Site（社區）**導入該活動

導入狀態：未導入 / 已導入 / 已停用

導入後效果：只有「已導入的 Site」，E-Life 才會顯示活動入口

導入/停用需可追溯：誰、何時啟用/停用（內部管理）

3.4 綁定成果追蹤（E-Life ↔ 好住家）

綁定數：總綁定、按 Site、按期間

綁定狀態查詢（客服支援）：未綁定 / 已綁定 / 綁定失敗（可簡述原因）
-（建議）綁定獎勵狀態（Open Point）：成功/待確認/失敗（避免住戶一直問）

3.5 消費成果追蹤（好住家 App 內消費）

已綁定住戶在活動期間於好住家消費的成果：筆數、金額、按 Site/期間

讓業務/行銷能回答：

哪些 Site 成效最好？

綁定多的 Site 是否也帶來更多消費？

檔期內哪段時間成效最高？

3.6 分潤報表（Revenue Report）

以 Site 為核心的分潤報表

常用維度：期間（日/週/月）、活動、Site

常用指標：綁定數、消費筆數/金額、分潤金額（或估算）
-（可選）匯出：提供財務彙整

1. 實體（名詞解釋與資料概念）

E-Life：你們的住戶 App（活動入口與綁定入口）

統一集團：合作廠商

好住家：統一集團的合作廠商 App（消費發生地、點數使用地）

Open Point：綁定成功後的獎勵點數（由統一集團提供/發放）

活動（Activity/Campaign）：一段期間內的合作規則集合（綁定送點＋消費分潤）

案場（Site）：導入活動的最小單位，也是分潤歸屬單位

導入（Enablement）：某活動是否在某 Site 生效（E-Life 是否顯示入口）

綁定（Binding）：住戶完成 E-Life 與好住家帳號連結（條件一）

消費（Purchase）：住戶在好住家 App 的消費行為（條件二）

分潤（Revenue Share）：你們因消費而取得的收益（按規則計算）

期間（Period）：報表彙總時間段（按月/週/日）

5. 流程（用故事線描述）
5.1 活動上線與導入（公司端）

你們建立「統一集團 × 好住家」合作活動（期間、說明、E-Life 入口素材）

指定要導入的案場 Site（例：A 社區導入、B 社區導入、C 不導入）

導入後，住戶在 E-Life（且屬於該 Site）才會看到活動入口

5.2 住戶綁定與領 Open Point（住戶端）

住戶在 E-Life 點活動入口 → 點「立即綁定」

完成 E-Life 與好住家帳號綁定

綁定成功後，住戶獲得 Open Point 點數（在好住家/統一集團體系中可用）

E-Life 顯示「已綁定」狀態（可選顯示點數已發放提示）

5.3 活動期間消費 → 分潤形成（成效與收益）

活動期間內，已綁定住戶在好住家 App 消費

後台累積消費成果（按 Site/期間彙總）

後台產出分潤報表（按 Site 歸屬、按期間彙總）

財務依報表彙整（必要時匯出）

5.4 活動結束後（回顧）

行銷/業務可回顧：哪些 Site 成效好、入口素材/檔期安排是否影響綁定與消費、下次導入策略

6. 需要先對齊的商務規則（非技術但非常關鍵）

Open Point 發放呈現：E-Life 是否要顯示「已發放」狀態？若住戶說沒收到，客服怎麼判斷？

活動期間認定：以哪一天/哪個時點為準（開始/結束）

分潤計算規則：比例/固定/階梯、是否封頂、是否排除特定品類/門市（若統一集團有要求）

退款/取消：若住戶後續退款，分潤是否要回沖？回沖算在哪個期間？

案場歸屬：分潤永遠以「消費發生時住戶所屬 Site」歸屬嗎？（通常是）
